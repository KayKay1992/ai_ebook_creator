## Building AI APIs (Generate Outline & Generate Chapter Content)

### Setup
- Uses `@google/genai` (Google's official Node.js SDK).
- A single `GoogleGenAI` instance is created at the top of the file using `GEMINI_API_KEY` from `.env`.
- Both routes are private — `router.use(protect)` in `aiRoute.js` covers all AI routes with JWT auth.

### Route Map — `routes/aiRoute.js`

| Method | Endpoint                          | Handler                 |
|--------|-----------------------------------|-------------------------|
| POST   | /api/ai/generate-outline          | generateOutline         |
| POST   | /api/ai/generate-chapter-content  | generateChapterContent  |

---

### 1. Generate Outline — `POST /api/ai/generate-outline`

**Input (req.body):** `topic` (required), `description`, `style`, `numChapters`

**What it does:**
1. Validates that `topic` is present — returns 400 if missing.
2. Builds a detailed prompt instructing Gemini to act as a "book outline generator". The prompt injects `topic`, `description`, `style`, and `numChapters` (defaults to 5 if not provided).
3. The prompt explicitly tells Gemini to return **only a raw JSON array** — no markdown, no extra text — where each object has a `title` and `description` key.
4. Calls `ai.models.generateContent({ model: 'gemini-2.5-flash-lite', contents: prompt })` and reads the text from `response.text`.
5. Since AI responses can sometimes include extra text around the JSON, it uses `text.indexOf('[')` and `text.lastIndexOf(']')` to extract just the JSON array portion.
6. Parses the extracted string with `JSON.parse()`. If parsing fails, returns 500. If it succeeds, returns `{ outline }` with status 200.

---

### 2. Generate Chapter Content — `POST /api/ai/generate-chapter-content`

**Input (req.body):** `chapterTitle` (required), `chapterDescription`, `style`

**What it does:**
1. Validates that `chapterTitle` is present — returns 400 if missing.
2. Builds a prompt instructing Gemini to write a full chapter (1500–2500 words) in plain text (no markdown). The prompt injects `chapterTitle`, `chapterDescription`, and `style` to shape the tone and content.
3. Calls `ai.models.generateContent({ model: 'gemini-2.5-flash-lite', contents: prompt })` and reads the plain text from `response.text`.
4. Returns `{ content: text }` directly — no JSON parsing needed since the output is plain prose, not structured data.

---

### Key Difference Between the Two

| | generateOutline | generateChapterContent |
|---|---|---|
| Output format | Structured JSON array | Plain text prose |
| Post-processing | Extract + parse JSON | Return text directly |
| Validation | topic required | chapterTitle required |

---

### Bugs Found & Fixed

1. **Wrong env variable name**
   - Was: `process.env.GOOGLE_API_KEY`
   - Fixed to: `process.env.GEMINI_API_KEY` (matching the actual key name in `.env`)

2. **Wrong SDK parameter key in `generateOutline`**
   - Was: `content: prompt` (singular — not a valid SDK parameter)
   - Fixed to: `contents: prompt` (correct key, consistent with `generateChapterContent`)
