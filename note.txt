Building Export APIs (Export as pdf or word document)

 In the exportController.js file, the exportAsPDF function is responsible for exporting a book as a PDF. It first checks if the book exists and if the user is authorized to export it. If the book is not found or the user is not authorized, it returns an appropriate error response.

The function then generates the PDF using the PDFDocument library. It creates a new PDF document and adds the necessary content, such as the cover page, chapters, and other elements. The cover page is added if there is an image available for the book. The chapters are added by iterating over the book.chapters array and rendering each chapter's content.

In the exportRoute.js file, the exportAsPDF and 
exportAsDocument
 routes are defined. These routes handle the requests to export the book as a PDF or a document, respectively. The exportAsPDF route is protected and requires authentication using the protect middleware.

In the server.js file, the app.use() middleware is used to handle the routes defined in exportRoute.js. The express.json() middleware is also used to parse the JSON data in the request body.

The changes made in server.js include adding the express.json() middleware and updating the routes to use the exportRoute.js file. This allows the /api/books/:id/pdf and /api/books/:id/doc routes to be handled by the appropriate functions in exportController.js.
