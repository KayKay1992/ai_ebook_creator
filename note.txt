Create eBook Popup Modal – Component Explanation & Implementation Flow
Overview
The Create eBook feature is implemented as a multi-step popup modal. Instead of navigating to a separate page, the user stays on the dashboard and a beautiful modal appears when they click “Create New eBook”. This creates a smooth, modern experience.

Main Components Involved
1. CreateBookModal
This is the core component. It manages the entire creation flow and contains two main steps:

Step 1 – Book Details
The user enters the book title, number of chapters, optional topic, and writing style. When they click “Generate Outline with AI”, the modal sends this information to the backend.
Step 2 – Review Outline
After the AI generates the outline, the user sees a list of chapters. They can edit chapter titles and descriptions, delete chapters, or add new ones. Once satisfied, they click “Create Book”.

The component also handles loading states, form validation, and resetting the form when the modal is closed.
2. Modal (Base Component)
A reusable wrapper that creates the dark backdrop, centers the content, and provides a clean header with a close button. It handles opening and closing animations and prevents the background from scrolling.
3. InputField & SelectField
These are shared form components used inside the modal. They provide consistent styling, icons, and focus states that match the rest of the application.
4. Button
A reusable button component that supports loading spinners and different visual styles (primary, secondary, etc.).

How the Flow Works

On the Dashboard page, a state variable (isCreateModalOpen) controls whether the modal is visible.
When the user clicks “Create New eBook”, the state is set to true, and the modal appears.
Inside the modal, the user fills out Step 1 and triggers the AI outline generation.
The frontend sends a request to the backend AI endpoint. The backend uses Gemini to create a structured list of chapters.
The response is received and the modal automatically switches to Step 2, displaying the generated chapters.
The user can refine the outline. When they click “Create Book”, the final data is sent to the backend to create a new book in the database.
After successful creation, the modal closes and the user is redirected to the book editor.


Design Approach
The modal follows the same premium design language used throughout the app:

Soft violet and purple accents
Large rounded corners
Clean progress indicator showing Step 1 and Step 2
Smooth transitions between steps
Hover effects and clear visual hierarchy

This creates a cohesive and professional experience that feels modern and polished.
