## Backend Setup & Project Structure

### 1. Initialized the Project
- Created the `backend/` folder and ran `npm init` to generate `package.json`.
- Set `"type": "commonjs"` since the backend uses `require/module.exports` (CommonJS).

### 2. Installed Dependencies
Core packages:
- `express` ‚Äî web framework for building the REST API
- `mongoose` ‚Äî ODM for interacting with MongoDB
- `dotenv` ‚Äî loads environment variables from `.env`
- `cors` ‚Äî handles Cross-Origin Resource Sharing
- `bcryptjs` ‚Äî hashes passwords before saving to DB
- `jsonwebtoken` ‚Äî generates and verifies JWT tokens for auth
- `multer` ‚Äî handles file uploads
- `@google/genai` ‚Äî Google Generative AI SDK for AI features
- `docx`, `pdfkit`, `markdown-it` ‚Äî eBook export utilities

Dev dependency:
- `nodemon` ‚Äî auto-restarts server on file changes during development

### 3. Entry Point ‚Äî server.js
- Loads `.env` variables via `dotenv.config()`.
- Creates an Express app and applies middleware:
  - `cors` configured to allow all origins with GET/POST/PUT/DELETE methods.
  - `express.json()` to parse incoming JSON request bodies.
- Serves uploaded files statically from the `uploads/` folder.
- Calls `connectDB()` to establish the MongoDB connection.
- Starts the server on `process.env.PORT` or defaults to port 5000.

### 4. MongoDB Connection ‚Äî config/db.js
- Exports an async `connectDB` function.
- Uses `mongoose.connect()` with the `MONGO_URI` from the `.env` file.
- Logs success or exits the process on connection failure.
