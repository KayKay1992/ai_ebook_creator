## Building eBook APIs (CRUD + Cover Upload)

All routes are private — every request must pass through the `protect` middleware first, which verifies the JWT token and attaches `req.user` to the request.

---

### Route Map — `routes/bookRoute.js`

| Method | Endpoint              | Handler         |
|--------|-----------------------|-----------------|
| GET    | /api/books            | getBooks        |
| POST   | /api/books            | createBook      |
| GET    | /api/books/:id        | getBookById     |
| PUT    | /api/books/:id        | updateBook      |
| DELETE | /api/books/:id        | deleteBook      |
| PUT    | /api/books/cover/:id  | updateBookCover |

`router.use(protect)` is applied once at the top, so every route in this file is automatically protected.

---

### 1. Create a Book — `POST /api/books`

- Pulls `title`, `author`, `subtitle`, `chapters` from the request body.
- Validates that `title` and `author` are present — returns 400 if missing.
- Calls `Book.create(...)` with `userId: req.user._id` so the book is always linked to the logged-in user.
- Returns the created book with status 201.

---

### 2. Get All Books — `GET /api/books`

- Runs `Book.find({ userId: req.user._id })` — only returns books that belong to the logged-in user. Other users' books are never exposed.
- Results are sorted by `createdAt: -1` (newest first).
- Returns the array of books with status 200.

---

### 3. Get One Book — `GET /api/books/:id`

- Finds the book by `req.params.id`.
- If not found → 404.
- Checks `book.userId.toString() !== req.user._id.toString()` — if the book belongs to a different user → 401 Unauthorized. This ownership check appears in every single-book operation.
- Returns the book with status 200.

---

### 4. Update a Book — `PUT /api/books/:id`

- Finds the book, runs the ownership check (same as above).
- Calls `Book.findByIdAndUpdate(id, req.body, { new: true })` — passes the entire request body as the update and returns the updated document (`{ new: true }`).
- Returns the updated book with status 200.

---

### 5. Delete a Book — `DELETE /api/books/:id`

- Finds the book, runs the ownership check.
- Calls `book.deleteOne()` to remove it from the database.
- Returns `{ message: 'Book removed' }` with status 200.

---

### 6. Update Cover Image — `PUT /api/books/cover/:id`

- Uses `uploadMiddleware` (multer) to handle the incoming file upload before this handler runs.
- Finds the book, runs the ownership check.
- If `req.file` exists, sets `book.coverImage` to the uploaded file path.
- If no file was uploaded → 400 "No file uploaded".
- Saves and returns the updated book.

---

### Ownership Check Pattern (used in every endpoint)

```js
if (book.userId.toString() !== req.user._id.toString()) {
    return res.status(401).json({ message: 'Unauthorized' });
}
```

Both sides are converted to strings before comparing because `userId` is a MongoDB ObjectId, not a plain string. This ensures one user can never read, edit, or delete another user's books.
