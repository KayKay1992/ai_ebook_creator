## Book Schema — models/Book.js

### Overview
The `Book` model defines the MongoDB document structure for an eBook. It is composed of two Mongoose schemas:
1. `chapterSchema` — a sub-document schema embedded inside `bookSchema`.
2. `bookSchema` — the top-level schema that represents the full eBook document.

---

### Dependencies
- `mongoose` — provides the schema/model layer on top of MongoDB.

---

### 1. chapterSchema (Sub-Document)

```js
const chapterSchema = new mongoose.Schema({
    title:       { type: String, required: true },
    description: { type: String, default: '' },
    content:     { type: String, default: '' },
});
```

A chapter is not a standalone MongoDB collection — it is embedded directly inside a book document as a **sub-document array**. Each chapter has:

| Field         | Type   | Required | Default | Notes                                         |
|---------------|--------|----------|---------|-----------------------------------------------|
| `title`       | String | Yes      | —       | The chapter heading / title                   |
| `description` | String | No       | `''`    | A short summary or description of the chapter |
| `content`     | String | No       | `''`    | The full written body/content of the chapter  |

- Because `chapterSchema` is defined without `{ _id: false }`, Mongoose automatically assigns a unique `_id` to each chapter sub-document. This allows individual chapters to be referenced or updated by their `_id`.
- `description` and `content` default to empty strings so a chapter can be created as a placeholder and filled in later (useful for AI-generated content workflows).

---

### 2. bookSchema (Top-Level Document)

```js
const bookSchema = new mongoose.Schema({
    userId:     { type: mongoose.Schema.Types.ObjectId, ref: 'User', required: true },
    title:      { type: String, required: true },
    subtitle:   { type: String, default: '' },
    author:     { type: String, required: true },
    coverImage: { type: String, default: '' },
    chapters:   [chapterSchema],
    status:     { type: String, enum: ['draft', 'published'], default: 'draft' },
}, { timestamps: true });
```

| Field        | Type       | Required | Default   | Notes                                                  |
|--------------|------------|----------|-----------|--------------------------------------------------------|
| `userId`     | ObjectId   | Yes      | —         | References the `User` who owns this book               |
| `title`      | String     | Yes      | —         | Main title of the eBook                                |
| `subtitle`   | String     | No       | `''`      | Optional subtitle                                      |
| `author`     | String     | Yes      | —         | Author name displayed in/on the book                   |
| `coverImage` | String     | No       | `''`      | URL or file path of the cover image                    |
| `chapters`   | Array      | No       | `[]`      | Array of embedded `chapterSchema` sub-documents        |
| `status`     | String     | No       | `'draft'` | Publication state; constrained to `draft` or `published` |

#### Field Details

1. **userId**
   - Type: `mongoose.Schema.Types.ObjectId` with `ref: 'User'`.
   - Creates a **reference (foreign key)** to the `User` collection.
   - Allows Mongoose's `.populate('userId')` to fetch the full user document when querying a book.
   - Required, so every book must be owned by a registered user.

2. **title / subtitle / author**
   - `title` and `author` are required strings — the minimum metadata needed to create a book.
   - `subtitle` is optional and defaults to `''`.

3. **coverImage**
   - Stores a URL or upload path (handled by `multer` via `uploadMiddleware.js`).
   - Defaults to `''` so the book can be created before a cover is uploaded.

4. **chapters**
   - Defined as `[chapterSchema]` — an array of embedded chapter sub-documents.
   - Stored directly inside the book document (no separate `chapters` collection).
   - Defaults to an empty array `[]` when no chapters are provided at creation time.
   - Each chapter carries its own auto-generated `_id`, `title`, `description`, and `content`.

5. **status**
   - Uses `enum: ['draft', 'published']` to restrict the value to only two valid states.
   - Defaults to `'draft'` so newly created books are not immediately visible as published.
   - Mongoose will throw a `ValidationError` if any other value is assigned.

---

### Schema Options

- `timestamps: true` — Mongoose automatically adds:
  - `createdAt` — when the book document was first created.
  - `updatedAt` — when the book document was last modified (including chapter edits).

---

### Model Export

```js
module.exports = mongoose.model('Book', bookSchema);
```

- Compiles the schema into a Mongoose model named `'Book'`.
- Mongoose maps this to the `books` collection in MongoDB (auto-pluralized, lowercased).
- Exported for use in controllers (`bookController.js`, `aiController.js`, `exportController.js`).

---

### Relationship Summary

```
User (users collection)
 └── has many Books (books collection)
        └── each Book embeds many Chapters (sub-documents, no separate collection)
```

- `User` → `Book` is a **referenced relationship** (via `userId` ObjectId ref).
- `Book` → `Chapter` is an **embedded relationship** (chapters live inside the book document).
